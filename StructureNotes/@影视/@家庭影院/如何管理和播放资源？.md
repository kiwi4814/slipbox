## 一、资源怎么存

### 1. 资源的存储和备份

首先说下资源的存储和备份，相信大家都会把重要的文件进行备份，备份的时候尽量选择平时使用不多的硬盘进行备份（是物理意义上的硬盘不是磁盘分区），比如可以选择平时不怎么插在电脑上的移动硬盘，也可以选择网盘，如果是敏感或者个人信息类的资料上传网盘的话不放心可以多压缩加密几次扔上去即可，很重要的资料可以做多地多份的容灾备份。

如果是影视资源进行备份的话（因为这类资源一般是能再找到的，所以一般不需要备份），尽量选择年代久远的、版本特殊的进行备份，因为BT下载是P2P，公网资源时间越久远也就越没人做种，你再从网上再找回来的可能性就越小。

### 2. 影视资源的版本



然后进入正题，资源怎么存，很多小伙伴都应该有私藏一些自己偏爱的以及不怎么好找的影视资源吧，这些资源可能放在自己的电脑硬盘、移动硬盘、NAS等存储服务器以及各式各样的网盘里，文件的目录和标题可能也是五花八门的，



### 3. 影视资源的命名规范

对于影视资源的分类，其实可以很简单的分为两种，电影和剧集，电影这个很好理解，电影、短片、动漫的剧场版这类讲述完整的一个故事的单个文件，相应的，剧集就是由一系列分集文件组成的连续剧，比如电视剧、动漫、纪录片等。



对于电影来说，一个电影文件的标题最核心的肯定是电影名，对于一些重名率比较高的名称，你最好还要注明电影的年份，所以一般电影的命名是电影名称+年份，年份前面加上空格并且用括号括起来：

```
/电影
   阿凡达 (2009).mkv
   银翼杀手2049 (2017).mkv
   银翼杀手 (1982).mkv
   银翼杀手 (1982) {edition-导演剪辑版}.mp4
```



此外，说几个不怎么常见的点，有些电影是有很多个版本的，比如哈利波特1和2的加长版，教父的导演剪辑版，这些在命名的时候可以加上额外的edition标注，然后如果你想保存影片的花絮、预告片或者其他电影的SP篇，也可以在后缀做特殊的标注：

```
/Movies
   /Avatar (2009)
      Avatar (2009).mkv
      Avatar (2009) {edition-Extended Cut}.mp4
      /Behind The Scenes
         Performance Capture.mkv
      /Deleted Scenes
         Bar Fight.mp4
         Lost Sister.mkv
      /Interviews
         Sigourney Weaver.mp4
         Stephen Lang.mp4
      /Scenes
         Arrival.mp4
      /Trailers
          Teaser Trailer.mp4
          Theatrical Trailer #1.mp4
          Theatrical Trailer #2.mp4
```



而对于比较专业的用户来说，比较标准的电影文件会在命名的时候附带上更多的信息，下面举个例子，比如一部电影的命名是：奥本海默.Oppenheimer.2023.UHD.BluRay.2160p.DTS-HD.MA.5.1.HEVC.REMUX-FraMeSToR.mkv，这里面我们只看标题其实就能看出很多信息出来，这是一部在2023年发行的电影《奥本海默》，此资源的分辨率是是4k UHD，视频基于蓝光原盘重新混制而成，音频格式是5.1声道的DTS-HD，压制组是FraMeSToR等。



对于电视剧来说则复杂一点，但最核心的几个要素还是一样，电视剧名称+年份+第几集，考虑到国外的很多电视剧是分季的，所以你需要标注这是第几季才行，季度的英文是Season，在文件中简写为S，集数的英文是Episode，在文件中简写为E或者Ep，所以电视剧的标准命名可以参考这个结构：

![image-20231114085604852](https://kiwi4814-1256211473.cos.ap-nanjing.myqcloud.com/img/image-20231114085604852.webp)

这里需要注意的是，对于一些国产剧是不会写季度信息的，因为就一季嘛，所以最好的办法就是批量加上S01的标题，对于一些网盘资源，不符合规范的就更多了，比如直接是1.mp4这样的文件表示第一集。这种资源如果要收的话，是需要做一些整理的，当然最好是批量做，这里推荐两个软件，分别是Windows版本的advanced renamer 和 Mac版本的A Better Finder Rename，这两个软件都有强大的重命名功能，支持批量替换、批量增加、正则表达式、自动递增等功能，当然了，最好是从根源上解决这个问题 ———— 收录更好版本的资源。

### 4. 影视资源的刮削

我们打开爱奇艺或者腾讯视频的时候，首先映入眼帘的除了广告肯定就是五颜六色的海报墙了，拿电影举例，这里包含了电影的海报、剧情简介、工作人员等信息，这些数据我们称之为一部电影的“元数据”，但这个元数据信息在我们自己上面整理的资源里当然是没有的，而所谓刮削，就是根据你的电影名去互联网数据库上下载元数据的过程，这样即使我们在浏览自己硬盘里的资源的时候，也能很方便的进行选片的操作，相当于实现了一个自己的爱奇艺。

![image-20231114183941586](https://kiwi4814-1256211473.cos.ap-nanjing.myqcloud.com/img/image-20231114183941586.webp)

影视资源的详细数据一般来自TMDB（The Movie Database），海报、预览图、图标等一般是图片的格式，而剧情介绍演员之类的文字信息一般存储在与电影同名的一个nfo文件中。



当然了，刮削我觉得作为一个概念了解下就可以了，大部分情况下在你资源命名规范的情况下是不需要自己去刮削的，因为你的视频播放器或者家庭影音服务会帮你做这个事情，你唯一需要的就是告诉他你的电影和剧集资源分别在哪个位置，他就会根据你的文件命名自动去互联网数据库检索和下载这些信息。当

然了，如果你说，我有强迫症，我就是要在本地刮削和保存这些电影海报剧情介绍等，不然就感觉不完整，也可以，你可以使用一款名为 tinyMediaManager（下面简称TMM） 的软件来进行刮削，此外这个软件也可以帮你整理和重命名文件，比如在刮削好电视剧后，按照自己定义的名称规范自动就把零散的文件整合命名，如果你对此软件很熟练，完全可以用它来替代你的影视资源文件管理。



在文件管理的部分之所以没推荐这个是因为它设置和功能比较复杂，具有一定的使用门槛，目前我一般用它来做资源的初始刮削和重命名，以及用来查找一些系列电影缺失的部分，剧集里没收录的部分（比如有些电视剧的特别篇以及动漫的OVA）。这个视个人兴趣可以尝试下，反正对于我来说，刮削的过程就是文件由杂乱到规整有序并且完整的一个过程，最终在电视或者其他终端呈现出来，很解压。

### 1.AppleTV + infuse

首先把AppleTV单独拿出来说一下吧，主要是目前的电视盒子里面，AppleTV确实是独一档的，原因除了本身超强的内核和解码能力之外，更重要的是一款叫 infuse 的软件，甚至可以说大部分用户买AppleTV的首要原因，颇有种”为了这碟醋才包的这顿饺子“的感觉。【当然，AppleTV上除了infuse外还有很多玩法，可以作为家庭中枢、旁路由以及IPTV等。】

![image-20231114184704233](https://kiwi4814-1256211473.cos.ap-nanjing.myqcloud.com/img/image-20231114184704233.webp)

infuse的本质其实就是一款附带了文件管理和影视刮削的视频播放器，可以连接家中处于同一网络连接之下的任何存储设备，最近的版本更新中新增的对于直连阿里云盘的支持更是让这个软件一度火出了圈。当然这不是全部的原因，软件本身的素质也是很能打的，无论是画质表现还是兼容性都是首屈一指的，无论是HDR还是杜比视界都支持的很好，此外，软件本身在苹果系全平台都有并且通过icloud同步数据，所以对于苹果全家桶玩家是很方便的。



下面是infuse在MacOS上打开后的效果，移动端和AppleTV电视上的显示笑果类似，可以参考下：

![image-20231114183439792](https://kiwi4814-1256211473.cos.ap-nanjing.myqcloud.com/img/image-20231114183439792.webp)



当然了，很多小伙伴平时喜欢用夸克、115、天翼等其他云盘，虽然infuse暂时不能直连，但也可以通过某些方式“曲线救国”的，这个可以我们待会儿再说。



所以，如果你比较“懒”但是又想要实现比较好的观影效果（影视墙、高码率、HDR、杜比视界等），性价比比较高的方式就是这个 AppleTV+infuse，甚至在使用阿里云网盘观影的情况下，不需要NAS也不需要额外的服务器，只要资源本身的命名是规范的（如上面所说，这个规范其实挺宽松的），大部分影视剧都能正确的被识别。



当然，infuse软件的细节很多，也并非上面说的那么十全十美，实际上也有很多令人诟病的毒点，目前市面上也已经出现了类似VidHub这样的竞品软件，相信在不久的将来，会有越来越多功能更好的软件出现，大家可以期待下。关于价格的话，infuse是订阅制的，想要尝试同款软件的话可以试试VidHub，国人开发，目前还不支持杜比视界（作者还在研究，等支持杜比了肯定要转成收费了）。



### 2.家庭影院三剑客

上面介绍了一种比较简单的方案，也是最容易看到效果的，但是正如上面所说，infuse的本质还是一个视频播放器，对于资源管理、分类和检索的部分是很薄弱的，举几个例子，播放列表、智能合集以及系列电影的自动聚合是很弱的，所以目前比较主流的方案是隐入一个“中间商”，使用单独的家庭影院管理服务来管理影音资源，而类似infuse这样的终端播放器只对接家庭影院服务来进行播放，各司其职。

目前关于家庭影院的服务，最主流的有三个，俗称家庭影院三剑客。分别是 Plex、 Emby 和 Jellyfin，后两者同源，Emby主要走收费路线提供更好的服务，Jellyfin则是开源的免费方案，Plex与他们的区别类似苹果系统和安卓系统的区别，闭源但很精致（另外也不便宜）。
